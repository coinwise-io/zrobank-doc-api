<h1 className={"openapi__heading"}>Add a new key to the user.</h1>

<MethodEndpoint method={"post"} path={"/pix/keys"}></MethodEndpoint>



Add a new key to the user. Allowed key types:
    <ul>
      <li>CPF: create a new key based on the document available in the user profile.
      <li>EMAIL: create a new key associated with an arbitrary e-mail. An e-mail will be sent with 5 digit code to validate it.
      <li>PHONE: create e new key associated with an arbitrary phone number. A SMS will be sent with 5 digit code to validate it.
      <li>EVP: create a random key.
      <li>CNPJ: create a new key based on the document available in the user profile.
    </ul>
    Return a created key which state is PENDING (types PHONE or EMAIL) or CONFIRMED (types CPF or EVP).
    <b>PS: The mobile app should call GET /pix/keys/{id}/code to send a code (e-mail or SMS) to the user.<b>
