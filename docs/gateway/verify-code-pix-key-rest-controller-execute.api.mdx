<h1 className={"openapi__heading"}>Confirm pending key code.</h1>

<MethodEndpoint method={"post"} path={"/pix/keys/{id}/code"}></MethodEndpoint>



Confirm code received by user due to adding/claim key request. This route works to PENDING and CLAIM_PENDING states.
    If key state is:
    <ul>
      <li>PENDING: key state will change to CONFIRMED.
      <li>CLAIM_PENDING: key state will change to CLAIM_DENIED and a claim cancel message will be sent to DICT.
    </ul>
    Return key which state is CONFIRMED and CLAIM_DENIED.
